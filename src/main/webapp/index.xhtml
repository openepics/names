Introduction

Format and syntax of the name components shall be as shown in the figure below. Only the device name is required on drawings, name tags, etc. where the drawing or device name clearly indicates the system and subsystem including the equipment. However, where the system and/or subsystem are not apparent, the full name must be shown.

Cryomodules are numbered sequentially by type (CA01, CA02…; CB01, CB02 …). Currently, there are 7 production types in FRIB, ranging from A to G. The warm sections following the accelerating cryomodules follow the cryomodules numbering (WA01, WA02…; WB01, WB02…). So CA01 is followed by WA01. Cavities, solenoids and phase detectors are numbered sequentially within a cryomodule (CAV1, CAV2, CAV3, …; SOL1, SOL2,…; PHD1, PHD2, ….). The input gate valve will be GV1, the output gate valve GV2 Beam Loss Monitors (BLM) go with the cryomodule or beam line name. In any case, the number behind the qualifier in the device name will uniquely define this BLM. MPS_CA01:BLM and MPS_BBS:BLM Examples for setting a cavity voltage: Set the voltage in the first cavity of the first cryomodule in Linac Segment 1: LS1_CA01:CAV1_D1093:V_CSET This same voltage may be set by addressing the LLRF via the name: C6SB _LLRF:CAV1_D1093:V_CSET (both will point to the same hardware address). A loop gain for this same LLRF system may be read by the rf specialist using: C6SB _LLRF:CAV1_D1093:G_RD Set the current of the solenoid in the eleventh cryomodule in Linac Segment 3: LS3_CD11:SOL1_D4712:I_CSET This same current may be set by addressing the PS via the name: C6SB_PS: PSOL_D4712:I_CSET (both will point to the same hardware address). Examples of commands:Switching a PS ON or OFF: C6SB_PS: PSOL_D4712:ON_CMD with value 1 (on) C6SB_PS: PSOL_D4712:ON_CMD with value 0 (off) Reset a PS: C6SB_PS: PSOL_D4712:RST_CMD with value 1 Open or close a gate valve GV: LS1_CB06:GV1_D1586:OPN_CMD with value 1 (open) LS1_CB06:GV1_D1586:OPN_CMD with value 0 (close) Examples of reading status: Read if ON (or OFF) CMD on a PS got accepted: C6SB_PS: PSOL_D4712:ON_CSTS, value 1=on, value 0=off Read actual PS status (ON or OFF): C6SB_PS: PSOL_D4712:ON_RSTS, value 1=on, value 0=off Read if open (or close) CMD on a GV got accepted: LS1_CB06:GV1_D1586:OPN_CSTS, value 1=open, value 0=close Read actual GV status (opened or closed): LS1_CB06:GV1_D1586:OPN_RSTS, value 1=opened, value 0=closed Reading a complete status word, containing multiple status bits: C6SB_PS: PSOL_D4712:MS_RD Setting multiple control bits simultaneously: C6SB_PS: PSOL_D4712:MC_CSET An example of the usage of CM (cryomodule) device name is the case whereby several signals from a cryomodule go to an IOC, which then creates one single interlock based on these signals (i.e. from the outside this is interlock is seen as one pertaining to the entire cryomodules): LS1_CA01:CM_D1105:ILK_RSTS Production target related devices will be named with FS_PTG as a system and sub-system name. Other devices in target systems, such as remote handling related ones, will use TGT as a system name. Equipment and associated “Tag Names” should be named according to IEEE 803, IEEE Recommended Practice for Unique Identification in Power Plants and Related Facilities, which references the Instrument Society of America (ISA) Standard S5.1 (“Instrumentation Symbols and Identification”). Instance numbering for signals (loops) should be as follows:
  • Power, communication & control systems: 0000 - 1999
  • HVAC systems: 2000 - 3999
  • Water systems: 4000 - 5999
  • Gas systems: 6000 - 7999
  • Waste systems: 8000 – 9999
  • Cryo systems: 10000 – 99999
